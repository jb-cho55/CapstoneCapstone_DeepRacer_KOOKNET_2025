# Reward Design

> KOOKNET DeepRacer Capstone 2025  
> 라운드별 보상함수 설계 의도와 개선 과정을 정리한 문서

---

## 1. 문서 목적

이 문서는 각 라운드에서 사용한 보상함수 코드를 단순히 나열하는 것이 아니라,

- 왜 그런 보상함수를 설계했는지
- 어떤 문제를 해결하려고 했는지
- 이전 모델의 한계가 무엇이었는지
- 다음 모델에서 무엇을 바꾸었는지

를 정리하기 위한 문서입니다.

DeepRacer 프로젝트에서 보상함수는 단순한 점수 계산식이 아니라,  
차량이 어떤 주행 습관을 학습할지를 결정하는 핵심 요소였습니다.

따라서 이 프로젝트에서는 “코드 자체”보다도  
**보상함수가 어떤 주행 행동을 유도하는지**를 중심으로 설계를 진행했습니다.

---

## 2. 설계 원칙

전체 라운드에 공통적으로 적용한 보상 설계 원칙은 다음과 같습니다.

### 1) 완주 가능성을 우선 확보한다
아무리 빠른 모델이라도 트랙을 자주 이탈하면 실제 대회에서는 불리했습니다.  
따라서 초기 설계에서는 최고 속도보다 **완주 가능성**을 우선했습니다.

### 2) 속도와 안정성을 함께 본다
속도만 높이면 랩타임은 빨라질 수 있지만,
코너에서 불안정해지거나 off-track이 증가할 수 있었습니다.  
반대로 안정성만 강조하면 지나치게 느려질 수 있었습니다.

즉, 보상함수는 항상  
**속도와 안정성의 균형**을 목표로 설계했습니다.

### 3) 잘못된 행동은 강하게 억제한다
특히 다음과 같은 행동은 빠르게 억제할 필요가 있었습니다.

- off-track
- 충돌
- 진행 방향과 크게 어긋난 주행
- 불필요하게 급격한 조향 변화

이런 경우에는 매우 작은 보상을 주거나 감점을 강화하는 방식으로 설계했습니다.

### 4) 좋은 행동은 연속적으로 유도한다
단순한 if/else 구조만 사용하면 보상이 급격하게 바뀌어
학습이 불안정해질 수 있었습니다.  
그래서 가능한 경우에는 중심선 거리, heading 차이, 장애물 거리 등을
**연속적으로 변화하는 보상값**으로 설계하려고 했습니다.

### 5) 실제 주행을 고려한다
시뮬레이션에서 잘 달리는 정책이 실제 차량에서도 그대로 재현되지는 않았습니다.  
그래서 프로젝트 후반부로 갈수록
**실차에서 반복 가능한 안정성**을 더 중요하게 보게 되었습니다.

---

## 3. Round 1 보상함수 설계

Round 1은 강화학습 기반 Time Trial 라운드로,  
가장 빠른 한 바퀴 기록이 목표였습니다.

하지만 실제 실험에서는 단순히 빠르게만 달리는 정책보다,
**오프트랙 없이 반복 가능한 정책**이 더 중요하다는 점을 확인했습니다.

### 3.1 Model 1 - 중심선 + 속도 기반

가장 기본적인 접근은 차량이 중심선에 가깝게 유지되면서
일정 속도 이상을 내도록 유도하는 것이었습니다.

주요 입력:
- `distance_from_center`
- `track_width`
- `speed`

설계 의도:
- 트랙 중앙을 따라 무난하게 주행
- 지나치게 느린 주행을 방지
- 가장 기본적인 reward baseline 확보

한계:
- 직선에서는 속도가 잘 나오지만 코너에서 불안정
- off-track이 자주 발생
- 빠른 기록 대비 재현성이 낮음

즉, 첫 번째 모델은  
“주행이 가능함”을 확인하는 baseline 역할은 했지만,
대회용 모델로는 안정성이 부족했습니다.

---

### 3.2 Model 2 - 오프트랙 강패널티

Model 1의 가장 큰 문제는 off-track이었기 때문에,
두 번째 모델에서는 이를 강하게 억제하는 방향으로 수정했습니다.

주요 입력:
- `all_wheels_on_track`
- `distance_from_center`
- `speed`

핵심 아이디어:
- `all_wheels_on_track == False` 이면 매우 작은 보상
- 중심선에서 멀어질수록 보상 감소
- 속도는 유지하되, 과도한 공격적 주행은 억제

설계 의도:
- 트랙 이탈을 학습 초기에 빠르게 줄이기
- 안정성을 우선 확보하기

장점:
- off-track 감소
- 평균적인 완주 성능 향상

한계:
- 속도가 낮아짐
- 지나치게 보수적인 주행
- 최고 랩타임 개선 효과는 제한적

즉, 이 모델은  
“빠르기보다 안전하게 달리는 정책”을 만드는 데는 효과적이었지만,
Time Trial 기준에서는 공격성이 부족했습니다.

---

### 3.3 Model 3 - 레이싱 라인 기반 보상

중심선 기반 접근의 한계를 보완하기 위해,
세 번째 모델에서는 레이싱 라인 기반 보상함수를 실험했습니다.

사용한 정보:
- `x`
- `y`
- `speed`
- `timeFromPreviousPoint`

설계 의도:
- 트랙의 기하학적 중심이 아니라
  실제 빠른 경로를 따라가도록 유도
- 구간별 적정 속도 반영
- 코너와 직선에서 다른 주행 전략 학습

장점:
- 최고 랩타임 자체는 개선 가능성 확인
- 공격적인 주행 전략 설계 가능

한계:
- 평균 성능 불안정
- 수렴 편차 존재
- 빠르지만 재현성이 낮은 결과 발생

즉, 레이싱 라인 기반 설계는 분명 잠재력이 있었지만,
보상 구조와 action space가 충분히 안정화되지 않으면
오히려 실전에서는 불안정할 수 있음을 보여주었습니다.

---

### 3.4 Final Model - 거리 + 방향 + 속도 스무딩

최종 모델은 앞선 세 모델의 장단점을 반영하여,
다단계 보상 구조로 설계했습니다.

핵심 구성:
1. 중심선 거리 보상
2. 진행 방향 정합성 보상
3. 속도 유지 및 스무딩 보상

#### 거리 기반 보상
- 차량이 트랙 내에서 안정적으로 유지되도록 설계
- 중심선에서 멀어질수록 보상 감소

#### 방향 정합성 보상
- 트랙 진행 방향과 차량 heading이 크게 어긋나지 않도록 유도
- 코너에서의 불안정성 완화

#### 속도 및 스무딩 보상
- 단순 고속 주행이 아니라
  급격한 steering 변화 없이 안정적으로 속도를 유지하도록 유도

설계 의도:
- 빠르지만 불안정한 모델이 아니라,
  실차에서도 어느 정도 재현 가능한 모델 만들기

핵심 교훈:
- Round 1에서는 결국
  **단일 목표(속도)보다 복합 목표(속도 + 안정성 + 방향성)**가 더 중요했습니다.

---

## 4. Round 2 보상함수 설계

Round 2는 Object Avoidance 라운드였습니다.  
즉, 차량이 트랙을 따라 달리는 것뿐 아니라
장애물을 피하는 행동까지 학습해야 했습니다.

여기서 가장 중요했던 점은  
**기본 주행 정책과 회피 정책은 별개의 문제**라는 것이었습니다.

### 4.1 Model 1 - 충돌 조건 추가

가장 처음에는 Round 1의 안정성 중심 보상함수 위에
충돌 조건만 추가하는 방식으로 시작했습니다.

주요 입력:
- `distance_from_center`
- `all_wheels_on_track`
- `is_crashed`

설계 의도:
- 기본 주행 성능 유지
- 충돌은 피해야 한다는 최소 조건 추가

한계:
- 충돌 후 패널티만으로는 회피 행동을 학습시키기 어려움
- 장애물을 미리 피하는 전략 형성이 부족
- 완주 실패 및 불안정성 존재

즉, “부딪히면 안 된다”는 사실만 알려주는 보상은
실제 회피 행동을 만들기에는 부족했습니다.

---

### 4.2 Model 2 - 거리 기반 위험 판단

두 번째 모델에서는 장애물과의 거리 정보를 적극 활용했습니다.

주요 입력:
- 중심선 거리
- 객체와의 거리
- LiDAR 거리 정보

설계 의도:
- 충돌 이후가 아니라 충돌 이전에 위험을 인식하도록 유도
- 장애물과 가까워질수록 감점을 강화
- 기본 주행과 회피를 동시에 고려

특징:
- 중심선 보상을 연속적인 함수 형태로 구성
- 객체 거리 기반 감점 추가
- 카메라 기반 판단 외에 LiDAR 정보 활용 시작

장점:
- 완주율 향상
- 충돌 감소 경향 확인

한계:
- 랩타임이 느림
- 회피 행동이 아직 충분히 자연스럽지 않음
- 일부 상황에서 불필요하게 보수적 주행

---

### 4.3 Model 3 - 단계적 학습 전략

Model 1, 2를 통해  
“주행과 회피를 처음부터 동시에 학습시키기 어렵다”는 점이 분명해졌습니다.

그래서 Model 3에서는 단계적 학습 전략을 도입했습니다.

#### 1단계
- 장애물 보상 제거
- Time Trial 중심으로 기본 주행 안정화

#### 2단계
- 1단계 학습 모델을 기반으로 clone
- 장애물 회피 관련 보상 추가
- 회피 기능을 후속 학습

설계 의도:
- 먼저 잘 달리게 만든 뒤,
  그 위에 장애물 회피 기능을 덧붙이기

핵심 의미:
- Round 2에서는 보상함수 내용뿐 아니라
  **학습 순서 자체도 설계 대상**이었습니다.

---

### 4.4 Model 4 - 통합형 회피 전략

최종 모델은 주행 안정성과 회피 성능을 동시에 반영하는 구조로 설계했습니다.

핵심 요소:
- 중심선 기반 안정성 보상
- 객체 거리 기반 감점
- 같은 차선 여부에 따른 패널티 차등
- LiDAR 기반 위험 거리 판단
- 회피 행동에 대한 추가 보상

설계 의도:
- 기본 주행 정책이 유지된 상태에서
  장애물에 가까워질수록 자연스럽게 회피 행동을 하도록 유도

LiDAR 활용 방식:
- 유효 거리값 추출
- 최소 거리(`min_lidar_distance`) 계산
- 전방 / 측면 거리 구분
- 일정 거리 이하에서 감점 강화

핵심 교훈:
- Round 2에서는 충돌 여부보다
  **충돌 가능성을 미리 예측하게 하는 보상 설계**가 중요했습니다.

---

## 5. Round 3 보상함수 설계

Round 3는 비공개 트랙 또는 새로운 트랙에 대응해야 했기 때문에,
두 방향의 설계를 병행했습니다.

- 범용 모델
- 특정 트랙 최적화 모델

### 5.1 범용 모델

범용 모델은 특정 트랙 최적화보다
일반화 성능을 우선했습니다.

주요 입력:
- `heading`
- `speed`
- 진행률 관련 정보
- off-track 여부

설계 의도:
- 트랙 진행 방향과 heading 정렬 유지
- 지나치게 공격적인 조향 억제
- 안정적인 완주 중심 보상
- 새로운 트랙에서도 무너지지 않는 기본 정책 형성

핵심 요소:
- heading alignment
- speed control
- completion bonus
- off-track penalty

장점:
- 새로운 트랙에서도 비교적 안정적
- 대회 당일 공개 트랙 대응 가능성 확보

한계:
- 특정 트랙 최고 기록은 한계 존재
- 공격적인 최적화 성능은 부족할 수 있음

---

### 5.2 A to Z Speedway 최적화 모델

A to Z Speedway에서는 레이싱 라인 기반 최적화를 진행했습니다.

사용 정보:
- `x`
- `y`
- `speed`
- `timeFromPreviousPoint`

설계 의도:
- 구간별 적정 속도 반영
- 빠른 경로 추종
- 랩타임 극대화

초기 한계:
- 완주율은 확보되었지만 불안정성 존재
- off-track이 간헐적으로 발생
- 평균 성능 편차 존재

원인 분석:
- 보상함수만이 아니라
  action space가 트랙 특성에 충분히 맞지 않았음

개선 방향:
- action space 조정 후 재학습
- 레이싱 라인 기반 보상과 제어 가능한 행동 공간의 정합성 확보

핵심 교훈:
- 특정 트랙 최적화에서는
  **보상함수와 action space를 함께 설계해야 한다**는 점이 중요했습니다.

---

### 5.3 Smile Speedway 최적화 모델

Smile Speedway 역시 레이싱 라인 기반으로 설계했지만,
여기서는 목표 랩타임 개념을 더 적극적으로 반영했습니다.

사용 정보:
- `x`
- `y`
- `speed`
- 예상 주행 시간

설계 의도:
- 빠른 경로를 따라가도록 유도
- 목표 랩타임 도달 시 추가 보상 부여
- 평균적인 성능 개선까지 함께 유도

학습 구조:
1. 레이싱 라인 기반 기본 학습
2. 목표 시간 달성 보상 추가

핵심 교훈:
- 특정 트랙에서는 단순한 경로 추종보다
  **시간 목표를 포함한 보상 설계**가 성능 향상에 유의미할 수 있었습니다.

---

## 6. 실패하거나 아쉬웠던 설계

프로젝트를 진행하며 잘 동작한 보상함수만 있었던 것은 아닙니다.  
오히려 실패한 설계에서 더 많은 교훈을 얻었습니다.

### 1) 속도만 과도하게 강조한 설계
- 직선에서는 빠르지만 코너에서 자주 무너짐
- off-track 증가
- 재현성 낮음

### 2) 오프트랙 패널티만 지나치게 크게 준 설계
- 지나치게 보수적인 주행
- 랩타임 저하
- 대회용 성능 부족

### 3) 충돌 이후 패널티만 주는 회피 설계
- 회피 행동 형성이 부족
- “부딪히지 않는 법”보다 “부딪힌 뒤 손해 보는 법”만 학습

### 4) 레이싱 라인만 믿고 action space를 충분히 조정하지 않은 설계
- 이론상 좋은 경로여도 실제 행동이 따라가지 못함
- 성능 편차 발생

---

## 7. 실차 주행에서 확인한 한계

보상함수는 시뮬레이션 안에서는 잘 동작해도,
실차 환경에서는 다음과 같은 차이가 존재했습니다.

- 차량별 조향 특성 차이
- 바닥 마찰 차이
- 캘리브레이션 상태
- 센서 노이즈
- 외부 환경 영향

이 때문에 프로젝트 후반부에는  
“가장 빠른 모델”보다
**“실제로 반복해서 안정적으로 달릴 수 있는 모델”**을 더 중요하게 보게 되었습니다.

즉, 보상함수 설계의 최종 목표는
시뮬레이션 점수 자체가 아니라
**실제 환경에서 재현 가능한 정책을 만드는 것**이었습니다.

---

## 8. 전체 프로젝트를 통해 얻은 교훈

이 프로젝트를 통해 얻은 핵심 교훈은 다음과 같습니다.

### 1) 보상함수는 주행 습관을 만든다
차량은 코드를 직접 이해하는 것이 아니라
보상 구조를 통해 “어떤 행동이 좋은지”를 학습합니다.

### 2) 하나의 좋은 수식보다 균형이 중요하다
특정 요소 하나를 강하게 최적화하는 것보다,
거리 / 방향 / 속도 / 안정성 / 회피를 균형 있게 보는 것이 더 중요했습니다.

### 3) 보상함수만으로 모든 문제가 해결되지는 않는다
실제로는 다음 요소도 매우 중요했습니다.

- action space
- 학습 순서
- 센서 해석 방식
- 실차 캘리브레이션
- 트랙 특성

### 4) 일반화와 최적화는 다른 문제다
새로운 트랙에서 안정적으로 달리는 정책과,
하나의 트랙에서 최고 성능을 내는 정책은
같은 목표가 아니었습니다.

### 5) 실전에서는 재현성이 중요하다
한 번 빠르게 달리는 모델보다
여러 번 반복해도 안정적으로 같은 성능을 내는 모델이 더 실용적이었습니다.

---

## 9. 향후 개선 방향

앞으로 추가로 개선한다면 다음과 같은 방향을 고려할 수 있습니다.

- 보상함수와 action space의 공동 최적화
- 트랙 유형별 보상 전략 분리
- sim-to-real 차이를 반영한 보정 구조 추가
- 센서 기반 회피 전략의 정교화
- 로그 기반 오프트랙/충돌 원인 분석 자동화
